# Руководство пользователя (черновик по скриншотам и действиям)

Этот файл подготовлен по структуре из примера: для каждого рисунка указано, что именно показать на скриншоте и как описать действия пользователя.

## 1. Аннотация

В документе описывается работа веб-приложения корпоративного обучения: вход в систему, работа по ролям (`EMPLOYEE`, `MANAGER`, `ANALYST`, `ADMIN`), прохождение курсов, тестов и заданий, а также административное управление сущностями системы и аналитика.

## 2. Назначение программы

- `EMPLOYEE`: просмотр назначенных курсов, прохождение уроков, тестов, выполнение заданий, просмотр результатов.
- `MANAGER`: создание и настройка курсов, уроков, тестов, заданий, назначение сотрудникам, проверка результатов.
- `ANALYST`: анализ показателей обучения, фильтрация данных, экспорт отчетов.
- `ADMIN`: управление ресурсами через админ-панель, резервное копирование и восстановление данных.

## 3. Условия выполнения программы

- ОС: Windows/Linux/macOS.
- Python и зависимости из `requirements.txt`.
- БД проекта (локальная `sqlite3` или PostgreSQL в зависимости от настроек).
- Современный браузер с доступом к серверу приложения.

## 4. Выполнение программы (рисунки и действия)

### Запуск и авторизация

1. **Рисунок 1 — Клонирование проекта и запуск сервера**
   Для начала работы пользователь разворачивает проект из репозитория, устанавливает зависимости и запускает сервер разработки командой `python manage.py runserver`. После успешного запуска в консоли отображается локальный адрес приложения, по которому открывается веб-интерфейс системы обучения.

2. **Рисунок 2 — Главная страница (`/`)**
   На главной странице пользователь видит центральную карточку приветствия «Система обучения и оценки сотрудников», краткое описание платформы и две основные кнопки: «Войти» и «Регистрация (через администратора)». Кнопка «Войти» переводит пользователя на страницу авторизации. Кнопка регистрации информирует, что самостоятельное создание учетной записи ограничено и выполняется через администратора. В правом верхнем углу расположен переключатель темы, позволяющий сменить оформление интерфейса между светлым и темным режимом.

3. **Рисунок 3 — Форма входа (`/accounts/login/`)**
   На странице входа пользователь вводит учетные данные в поля `Email` и `Пароль`, после чего нажимает кнопку «Войти». При корректных данных система создает пользовательскую сессию и перенаправляет в рабочий раздел согласно роли учетной записи. При ошибке авторизации на форме отображается сообщение «Неверный email или пароль», и пользователь остается на странице для повторного ввода данных. Внизу формы размещена ссылка «Регистрация» для перехода к созданию учетной записи, если это разрешено в текущем процессе эксплуатации.

4. **Рисунок 4 — Форма регистрации (`/register/`)**
   На странице регистрации пользователь заполняет поля формы (персональные и учетные данные), затем нажимает «Зарегистрироваться». Если данные проходят валидацию, создается новая учетная запись, и пользователь получает возможность войти в систему. Если обязательные поля заполнены некорректно или нарушены ограничения уникальности, система выводит ошибки формы и предлагает исправить введенные значения без потери контекста страницы.

5. **Рисунок 5 — Профиль пользователя (`/profile/`)**
   После входа пользователь может открыть страницу профиля, где отображаются ФИО, роль, email, подразделение/должность, а также текущий аватар. В блоке «Настройки профиля» доступны поля для редактирования личных данных и загрузки изображения. После изменения информации пользователь нажимает «Сохранить», а для завершения работы использует кнопку «Выйти», которая закрывает активную сессию и возвращает на экран авторизации.

### Сценарии сотрудника (EMPLOYEE)

6. **Рисунок 6 — Мои курсы (`/employee/`)**
   После входа сотрудник открывает раздел «Мои курсы», где отображается таблица всех назначенных ему программ обучения. Для каждой записи показываются наименование курса, назначенные сроки и текущий статус прохождения. В правой части строки доступна кнопка «Открыть», по которой пользователь переходит к детальной странице выбранного курса. Этот экран используется как стартовая точка работы сотрудника в системе.

7. **Рисунок 7 — Страница курса сотрудника (`/employee/courses/<id>/`)**
   На странице курса сотрудник видит карточку с названием и описанием, а ниже — рабочие блоки с учебными сущностями: «Модули», «Доступные тесты», «На пересдаче», «Пройденные тесты», «Задания». Из этого экрана пользователь управляет всем процессом прохождения: изучает уроки, запускает тесты, принимает решения по пересдаче и открывает назначенные задания. Для возврата к общему списку используется кнопка «К списку курсов».

8. **Рисунок 8 — Блок «Модули» на странице курса**
   В блоке «Модули» отображается список опубликованных уроков курса. По каждому уроку пользователь видит его название и может перейти к содержимому по кнопке «Открыть». При переходе сотрудник попадает на страницу конкретного урока, где доступно последовательное изучение материалов по страницам.

9. **Рисунок 9 — Страница урока (`/employee/lessons/<id>/`)**
   На странице урока отображаются заголовок, краткое описание и кнопка возврата к курсу. Если урок состоит из нескольких страниц, в верхней части контента появляются переключатели страниц; при нажатии на нужную вкладку содержимое обновляется без перехода на другой URL. Такой режим позволяет последовательно проходить материал в рамках одного экрана.

10. **Рисунок 10 — Контентные блоки урока**
    Внутри страницы урока сотрудник работает с блочной структурой контента: текстовые и инструкционные блоки, изображения, видео, ссылки на документы, кнопки перехода и встроенные блоки тестов. Пользователь может просматривать медиафайлы, открывать вложенные документы, переходить по кнопкам действий и запускать тест прямо из урока, если тест привязан к соответствующему блоку.

11. **Рисунок 11 — Доступные тесты на странице курса**
    В блоке «Доступные тесты» сотрудник видит перечень тестов, которые еще не завершены, с указанием проходного балла, дедлайна и количества использованных/доступных попыток. Для каждого теста доступна кнопка «Пройти». Нажатие переводит пользователя на экран прохождения теста.

12. **Рисунок 12 — Прохождение теста (`/employee/tests/<id>/take/`)**
    Экран тестирования содержит последовательность вопросов и элементы ответа в зависимости от типа вопроса: одиночный выбор, множественный выбор, сопоставление, упорядочивание, короткий или развернутый текстовый ответ (для развернутого ответа допускается вложение файла). После заполнения ответов сотрудник нажимает «Завершить тест». Система проверяет полноту и корректность заполнения, затем фиксирует попытку и переводит пользователя на страницу результата.

13. **Рисунок 13 — Результат теста (`/employee/tests/results/<id>/`)**
    На странице результата отображаются статус попытки (например, «Ожидает проверки», «Проверено · Пройден», «Проверено · Провален», «На пересдаче»), итоговый процент при наличии автопроверки, а также комментарий менеджера, если он уже оставлен. Ниже показана расшифровка ответов по каждому вопросу, включая выделение правильных и ошибочных элементов. Этот экран используется для анализа ошибок перед следующей попыткой или для подтверждения успешного прохождения.

14. **Рисунок 14 — Блок «На пересдаче»**
    Если менеджер перевел попытку в статус «На пересдаче», тест появляется в отдельном блоке «На пересдаче» на странице курса. В этой строке сотруднику доступны действия «Принять» и «Отказаться». При принятии пересдачи система разрешает повторное прохождение теста (с учетом настроек лимита попыток), при отказе тест переводится в статус отказа, и повторная попытка блокируется.

15. **Рисунок 15 — Блок «Задания»**
    В блоке «Задания» сотрудник видит назначенные практические работы по курсу. Для каждой работы отображается название, срок и текущий статус выполнения. По кнопке «Открыть» пользователь переходит на карточку конкретного задания для просмотра условий и отправки решения.

16. **Рисунок 16 — Страница задания (`/employee/tasks/<id>/`)**
    На странице задания сотруднику доступны все данные, необходимые для выполнения: тип задания, максимальный балл, срок сдачи, описание, критерии и прикрепленные материалы. Ниже расположена форма сдачи с полем текстового ответа и полем загрузки файла. После отправки система сохраняет новую или обновленную сдачу и фиксирует событие выполнения задания в истории обучения.

17. **Рисунок 17 — Комментарий менеджера по заданию**
    После проверки отправленного задания на этой же странице появляется блок «Комментарий менеджера», где выводятся оценка и текстовая обратная связь. Сотрудник использует этот блок для понимания качества выполненной работы и, при необходимости, вносит доработки в ответ с последующей повторной отправкой.

### Сценарии менеджера (MANAGER)

18. **Рисунок 18 — Панель менеджера (`/manager/`)**
    Что показать: кнопки быстрого доступа (`Создать курс`, `Мои курсы`, `Создать модуль`, `Создать тест`, `Создать задание`).
    Действия пользователя: выбрать сценарий управления контентом.

19. **Рисунок 19 — Навигация менеджера (верхнее меню)**
    Что показать: пункты `Панель`, `Курсы`, `Модули`, `Назначение заданий`, `Список учащихся`, `Черновики`.
    Действия пользователя: перейти в нужный раздел управления.

20. **Рисунок 20 — Список курсов (`/manager/courses/`)**
    Что показать: карточки курсов и действия `Открыть курс`, `Изменить`, `Назначить`, `Удалить`.
    Действия пользователя: выбрать курс для настройки.

21. **Рисунок 21 — Создание/редактирование курса**
    Что показать: поля `Название`, `Специальность`, `Обложка`, `Описание`, `Активен`.
    Действия пользователя: заполнить форму и сохранить курс.

22. **Рисунок 22 — Карточка курса (`/manager/courses/<id>/`)**
    Что показать: обложку, статус активности, кнопки `Изменить курс`, `Назначить сотрудников`, `Создать тест`, `Создать задание`.
    Действия пользователя: перейти к управлению участниками и сущностями курса.

23. **Рисунок 23 — Назначение сотрудников на курс**
    Что показать: поля срока, чекбокс `Выбрать всех`, список сотрудников.
    Действия пользователя: выбрать сотрудников и назначить курс.

24. **Рисунок 24 — Список модулей/уроков в курсе**
    Что показать: таблицу уроков и действия `Изменить`, `Удалить`, `Создать урок`.
    Действия пользователя: управлять структурой уроков.

25. **Рисунок 25 — Конструктор урока (`/manager/lessons/create|edit/`)**
    Что показать: боковую панель блоков (`Текст`, `Инструкция`, `Изображение`, `Документ`, `Видео`, `Кнопка`, `Тест`), вкладки страниц, редактор.
    Действия пользователя: собрать структуру урока по страницам и блокам.

26. **Рисунок 26 — Управление тестами в курсе**
    Что показать: список тестов и кнопки `Изменить`, `Вопросы`, `Проверить`, `Опубликовать/Отозвать`, `Удалить`.
    Действия пользователя: публиковать тесты и контролировать жизненный цикл.

27. **Рисунок 27 — Создание теста (`/manager/tests/create/`)**
    Что показать: поля параметров теста (название, курс, сроки, проходной балл, попытки, режим проверки).
    Действия пользователя: создать тест и подготовить его к наполнению вопросами.

28. **Рисунок 28 — Вопросы теста (`/manager/tests/<id>/questions/`)**
    Что показать: выбор типа вопроса (`SINGLE`, `MULTI`, `MATCHING`, `ORDERING`, `SHORT`, `LONG`), формы вариантов/пар/порядка.
    Действия пользователя: добавить вопросы и правильные ответы.

29. **Рисунок 29 — Проверка попыток (`/manager/tests/<id>/results/`)**
    Что показать: таблицу попыток сотрудников, статусы и кнопку `Проверить`.
    Действия пользователя: открыть конкретную попытку для проверки.

30. **Рисунок 30 — Экран проверки попытки (`/manager/tests/results/<id>/`)**
    Что показать: форму решения (`Статус`, `Результат`, `Комментарий`) и ответы сотрудника.
    Действия пользователя: выставить итог и сохранить результат проверки.

31. **Рисунок 31 — Назначение заданий (`/manager/tasks/`)**
    Что показать: список заданий и действия `Изменить`, `Выбрать сотрудников`, `Удалить`.
    Действия пользователя: выбрать задание для назначения сотрудникам.

32. **Рисунок 32 — Создание/редактирование задания**
    Что показать: поля задания (тип, курс, описание, критерии, срок, вложение, публикация).
    Действия пользователя: сформировать задание и сохранить.

33. **Рисунок 33 — Назначить задание (`/manager/tasks/<id>/assign/`)**
    Что показать: срок выполнения и список сотрудников для назначения.
    Действия пользователя: назначить задание выбранным сотрудникам.

34. **Рисунок 34 — Список учащихся (`/manager/students/`)**
    Что показать: список сотрудников и кнопку `Профиль`.
    Действия пользователя: открыть карточку сотрудника.

35. **Рисунок 35 — Профиль сотрудника (`/manager/students/<id>/`)**
    Что показать: персональные поля сотрудника и кнопку сохранения.
    Действия пользователя: скорректировать данные сотрудника.

36. **Рисунок 36 — Черновики (`/manager/drafts/`)**
    Что показать: таблицы черновиков тестов и заданий с кнопками `Опубликовать`, `Изменить`, `Удалить`.
    Действия пользователя: перевести сущность из черновика в опубликованное состояние.

37. **Рисунок 37 — Материалы (`/manager/materials/`)**
    Что показать: фильтр по курсу, список лекций, действия `Изменить`/`Удалить`.
    Действия пользователя: управлять библиотекой лекционных материалов.

38. **Рисунок 38 — Редактор материала (`/manager/materials/create|edit/`)**
    Что показать: WYSIWYG-панель (`Ж`, `К`, `Ч`, `Список`, `H2`, `Цитата`, `Ссылка`, `Картинка`) и поле содержимого.
    Действия пользователя: подготовить и сохранить материал.

## 5. Сообщения оператору (шаблон для твоей таблицы)

Сделай в отчете отдельную таблицу в формате примера:

- №
- Текст сообщения
- Содержание сообщения
- Действия оператора

Минимально включи эти группы сообщений:

- Авторизация: неверный логин/пароль, успешный вход, выход.
- Профиль: успешное обновление, ошибки валидации.
- Курсы/назначения: курс создан/обновлен/удален, сотрудники назначены, нет доступных сотрудников.
- Уроки/материалы: урок или материал создан/сохранен, ошибка загрузки файла.
- Тесты: опубликован/отозван, попытка отправлена, требуется ручная проверка, перевод на пересдачу.
- Задания: задание создано/назначено, работа отправлена, комментарий менеджера сохранен.
- Аналитика: ошибка фильтра даты, успешный экспорт XLSX/PDF.
- Админ-панель: запись создана/обновлена/удалена, бэкап создан, ошибка восстановления.

## 6. Как использовать этот черновик

1. Идешь по пунктам 1–45.
2. Для каждого пункта делаешь 1 скрин именно указанного экрана.
3. Под каждым рисунком вставляешь абзац из пункта (действия пользователя + назначение экрана).
4. В конце добавляешь таблицу сообщений оператора по разделу 5.
